3. 每个用户的隐式意图列表("implicit_needs")中包含2项待生成内容，不要私自增减列表项数量。其中隐式意图描述("need")为用户在初始询问中未明确提及，但在当前话题的讨论中交互助手应当理解到的用户诉求，每一项隐式意图描述仅聚焦在一个方面，需要用简短的话概括。相对应的依据（"evidences"）则为一个列表，列出用户背景和经历中反映该项隐式意图的各条细节，该部分列表可包含1~5项内容。注意依据（"evidences"）中的每一项只给出相关的用户背景和经历细节本身即可，不需要描述用户经历对应的具体日期，也不要包含额外的分析或推测过程。此外，用户的隐式意图列表中的每一项隐式意图仍应紧密围绕当前话题指定的需求类型，并确保能够共同形成一条较为聚焦的真实需求，而非反映用户的多方面不同需求。

5. 输出内容中不要出现用户姓名，用“用户”指代即可。
