# requirement analysis
- 输入
    - 对话上文
    - 待更新的需求描述
    - 当前轮次
- 输出
    - 需求更新分析（是否更新、更新后内容）

prompt：
````
【system prompt】
你是一个个性化对话助手，能够理解用户在对话中的个性化需求。
【user prompt】
你正在与用户进行对话交互，交互过程可能包含用户需求以及针对当前需求的解决方案讨论过程。输入中将会给出用户与你的对话上下文，需要你对于对话中用户表现出的需求进行总结。

# 对话上下文
以下为当前用户("user")与你("assistant")的对话上下文。
"""
- "user": "最近工作特别忙，有没有什么办法可以让我更高效地完成每天的工作？",
- "assistant": "听起来你希望找到一种适合自己的时间规划工具，对吗？我注意到你最近开始每天早晨列出核心任务清单，这说明你在尝试优化工作规划。"
- "user": "是的，我确实需要一个能帮我合理规划任务的东西，不然总是会被琐碎的事情打乱节奏。",
- "assistant": "另外，我猜你也希望能尽量减少外部干扰，以便集中精力处理重要任务，比如之前因为临时会议错过午餐的情况。"
- "user": "没错，突发性事务经常让我分心，导致效率很低。所以我想知道有没有什么具体的方法解决这个问题。",
- "assistant": "你可以试试用Notion或Todoist这样的数字化工具创建动态任务清单，并设置提醒功能，这样即使出差也能随时更新和管理任务。"
- "user": "这些工具听起来不错，但它们会不会太复杂了？我平时已经很忙了，没时间学习怎么用新软件。",
- "assistant": "<待生成>"
"""

# 输出要求及输出模板
## 输出要求
1. 仿照下面的输出模板，以json格式给出输出。用户需求("requirement")采用较简洁的一句话的形式，概括描述交互过程中用户想要寻求建议的具体需求内容。
2. 在需求描述中，不需要关注用户对于解决方案的偏好。
3. 除要求的json格式输出外，不要输出任何其它内容，也不需要在json中添加注释进行额外说明。

## 输出模板
"""
{
    "requirement": "..." // 当前话题下的用户需求描述，采用较简洁的一句话概括
}
"""

现在，请你按照输出要求和模板给出输出。
````
