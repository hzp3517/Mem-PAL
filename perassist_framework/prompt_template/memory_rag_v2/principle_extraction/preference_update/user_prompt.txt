## 具体要求
请你判断当前新增的偏好经验描述是否已经符合给出的用户偏好准则。如果新增的偏好经验描述与给出的准则存在冲突，或现有的准则明显遗漏了当前经验所体现的一些重要内容，则需对于用户偏好准则进行更新。
注意：
1. 现有的用户偏好准则已经是对于交互历史中相似偏好经验的概括，因此在非必要的情况下应尽量维持原来的用户偏好准则不变。
2. 如果需要对原准则进行调整，应综合考虑原准则内容和新增经验中体现的内容两方面进行更新。更新后的用户偏好准则仍需满足以下要求：是较为简洁的一句话形式，是对具体经验的进一步抽象和概括，且应是对应于输入中的“需求类型”所展开。
3. 仿造下面的输出示例，以json格式输出是否需要调整准则的判断，以及更新后的准则内容（如果需要调整）。
4. 除要求的json格式输出外，不要输出任何其它内容，也不需要在json中添加注释进行额外说明。
