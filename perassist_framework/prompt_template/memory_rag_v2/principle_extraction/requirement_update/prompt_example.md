## 具体要求
请你判断当前新增的需求内容是否已经属于给出的总体需求类型。如果新增的需求内容与给出的总体需求类型存在冲突，则需对于总体需求类型描述进行更新。
注意：
1. 现有的总体需求类型描述已经是对于交互历史中相似需求内容的概括，因此在非必要的情况下应尽量维持原来的总体需求类型描述不变。
2. 如果需要对原来的总体需求类型描述进行调整，应综合考虑原总体需求类型描述和新增的需求内容两方面进行更新。更新后的总体需求类型描述仍需满足以下要求：是一个较为简洁的短句形式；不要包含具体的细节信息（特别是不要出现具体的例子），而应从较为抽象的宏观层面描述总体需求类型；不需要包含用户偏好信息。
3. 仿造下面的输出示例，以json格式输出是否需要调整原总体需求类型的判断，以及更新后的总体需求描述（如果需要调整）。
