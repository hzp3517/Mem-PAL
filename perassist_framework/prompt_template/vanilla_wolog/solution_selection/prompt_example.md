# solution selection
- 输入
    - 当前用户需求
    - 候选方案列表
- 输出
    - 方案偏好分析

prompt：
````
【System Prompt】
你是一个个性化助手，能够针对用户的需求给出符合其个性化偏好的解决方案。
【User Prompt】
你正在与用户进行交互，给你当前的用户需求描述，同时给出针对当前需求的8条候选解决方案建议，请你深入理解用户的个性化偏好，并从候选方案建议中选出2条最符合用户偏好的方案。

# 当前用户需求
"""
用户希望通过制定灵活且可行的运动计划，并结合健康饮食的具体指导，有效控制体重增长，同时改善身体健康状况。
"""

# 候选方案建议
以下为针对当前用户需求的8条候选方案，包括各条方案的id和方案的内容。
"""
{
    'S1': "每周规划3-4次短时间高强度间歇训练（HIIT），以适应忙碌的日程安排。",
    'S2': "尝试步行上下班或午休时散步，增加日常活动量。",
    'S3': "利用酒店健身房或房间内的简单器械进行出差期间锻炼。",
    'S4': "采用“手掌法则”控制饮食分量，避免摄入过多热量。",
    'S5': "准备健康便携零食（如坚果、水果）应对不规律用餐情况。",
    'S6': "减少精制糖和高脂肪食品的摄入，优先选择全谷物和优质蛋白质。",
    'S7': "每餐保证蔬菜占比超过一半，提高膳食纤维摄取量。",
    'S8': "下载健身App跟踪运动数据，并设置提醒保持规律性。"
}
"""

# 输出模板
以下为json格式的输出模板，"solution"为待生成的方案内容。
"""
{
    "analysis": "...", // 用一段话给出对于所有候选方案的整体分析，关注于用户的个性化偏好
    "selected_solutions": [...] // 列出2个最符合用户偏好的方案id
}
"""

# 输出要求
1. 深入理解用户的个性化偏好，并基于用户偏好给出对于候选方案的分析和选择。
2. 遵循上述输出模板给出输出，其中"selected_solutions"部分只能选择最符合偏好的2项方案id作为输出，不能包含更多或更少的方案id。
3. 除要求的json格式输出外，不要输出任何其它内容，也不需要在json中添加注释进行额外说明。

现在，请你按照输出要求和模板给出输出。
````
